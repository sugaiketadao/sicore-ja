# HTML・CSS コーディング規約

## 概要
- HTML と CSS のコーディング規約を定義します。
- 本規約に準じることで、プログラムコードの統一化と開発・保守の効率化を目指します。

## 前提条件
- [Webページ構成標準](../02-develop-standards/01-web-page-structure.md) に従う。

## name属性名・data-name属性名・id属性名・CSSクラス名
- `name`属性名はロワースネークケースとする。一覧部分のみ `.` の使用を許可する。（[Webページ構成標準 - name属性](../02-develop-standards/01-web-page-structure.md#name属性) を参照すること。）
    - ［例１］ `name="lower_snake_case"`
    - ［例２］ `name="list.lower_snake_case"`
- `data-name`属性名の命名規約は、`name`属性と同じ規約とする。
- `id`属性名はロワーキャメルケースとする。
    - ［例］ `id="lowerCamelCase"`
- CSSクラス名は小文字のケバブケースとする。
    - ［例］ `class="lower-kebab-case"`
- アンダースコア `_` で始まる `name`属性名、`id`属性名、CSSクラス名は本フレームワークが使用するため、アプリケーションでの使用を禁止する。

## HTMLスタイル
- 要素を入れ子構造で記述する場合のインデントは、半角スペース 2文字を使用し、タブ文字は使用しない。
- タグ名は小文字を使用する。
    - ［OK 例］ `<input>`
    - ［NG 例］ `<INPIT>`
- 属性名は小文字を使用し、属性値はダブルクォーテーションで囲む。
    - ［OK 例］ `type="text"`
    - ［NG 例］ `TYPE='TEXT'`
- input 要素は閉じない。
    - ［OK 例］ `<input type="text">`
    - ［NG 例］ `<input type="text"/>`
- checked, disabled, readonly 等のブール属性は属性名のみ記述する。
    - ［OK 例］ `<input readonly>`
    - ［NG 例］ `<input readonly="readonly">`
- 属性のイコールの前後にはスペースを入れない。
    - ［OK 例］ `type="text"`
    - ［NG 例］ `type = "text"`
- 要素内の文字列リテラルの前後で不要な改行を入れない。また、改行を入れる場合は不要なインデントを入れない。
```［OK 例］
<label>［OK 例］<br>
要素内の文字列リテラルの前後で不要な改行を入れない。<br>
また改行をいれる場合は不要なインデントを入れない。<label>
```
```［NG 例］
<label>
    ［NG 例］<br>
    要素内の文字列リテラルの前後で不要な改行を入れない。<br>
    また改行をいれる場合は不要なインデントを入れない。
<label>
```

## CSSスタイル
- ブレース内のインデントは、半角スペース 2文字を使用し、タブ文字は使用しない。
- ブレース `{` の直後と `}` の直前に改行する。
- 文字列はダブルクォーテーションで囲む。
- 下記の箇所には半角スペース 1文字を入れる。ただし、行末や行末のセミコロンの直前には入れない。
    - 子セレクタ `>` や隣接セレクタ `+` などの前後。
    - 演算子（`+`, `-` など）の前後。
    - コロンとその後ろにある文字列の間。
    - `!important` とその前にある文字列の間。
- 数値ゼロの単位は省略する。

## コメント
- HTMLタグや CSSクラスはコメントアウトせず、削除する。注意点がある場合は、注意点のみコメントとして残す。
- 変更履歴のコメントは記載しない。注意点がある場合は、注意点のみコメントとして残す。
