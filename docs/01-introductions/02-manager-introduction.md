# SICoreフレームワーク マネージャー向け紹介

本資料では、開発チームへの導入を検討しているマネージャー向けに、フレームワークのメリットを解説します。

---

## 1. このフレームワークでできること

### 画面機能とAPI機能を同じ仕組みで開発

```
ブラウザ             サーバー
  │                   │
  │------------------>│
  │　　　　　　  Webサービス処理
  │<------------------│

サーバー             サーバー
  │                   │
  │------------------>│
  │　　　　　　  APIサービス処理
  │<------------------│
```

- 画面からの操作もAPIからの呼び出しも**同じプログラム**で対応できる。
- モバイルアプリや外部システムとの連携が容易である。

---

## 2. 開発効率が高い理由

### 2.1 プロトタイプがそのまま本番になる

| 従来の開発 | このフレームワーク |
|-----------|------------------|
| デザイン画面を作成する。 | デザイン画面を作成する。 |
| プログラム用に作り直す。 | **そのまま使用できる。** |
| 動作確認・修正を行う。 | 動作確認・修正を行う。 |

**効果**: デザインからプログラムへの変換作業が不要である。

### 2.2 設定ファイルが少ない

| 従来の開発 | このフレームワーク |
|-----------|------------------|
| 複数の設定ファイルが必要である。 | 最小限の設定で済む。 |
| URL設定が必要である。 | URL設定が不要である（自動）。 |
| DB接続設定が複雑である。 | DB接続設定のみで済む。 |

**効果**: 設定ミスによるトラブルが減少する。

### 2.3 書き方が統一される

- フレームワークがコードの書き方を強制する。
- 誰が書いても同じような構造になる。
- 引き継ぎや保守が容易である。

---

## 3. AIとの相性

### AIがプログラムを作成しやすい設計

このフレームワークは**AI（生成AI）やプログラミング初心者**に適した設計になっています。

| 特徴 | AIにとってのメリット |
|------|---------------------|
| ルールが明確である。 | 迷わずコードを生成できる。 |
| パターンが統一されている。 | 過去の例を参考にしやすい。 |
| 設定ファイルが少ない。 | 設定ミスが起きにくい。 |
| エラー対策が組み込み済みである。 | バグが入りにくい。 |
- AIに「検索画面を作って」と指示するだけで、動くコードを生成できる。
- 既存のサンプルを渡せば、類似機能を自動作成できる。

---

## 4. 安定性・信頼性

### 4.1 外部ライブラリに依存しない
外部ライブラリへの依存を極力排除しています。Tomcatも不要です。

| 項目 | 内容 |
|------|------|
| 使用技術 | 標準的なJava、HTML、JavaScript、CSSを使用する。 |
| 外部依存 | DB や Excel、PDF などの特殊処理のみに限定する。 |
| 更新リスク | 低い（外部ライブラリの更新影響が小さい）。 |

**効果**: 
- セキュリティパッチ対応が容易である。
- 長期間安定して運用可能である。

### 4.2 バグが入りにくい仕組み

フレームワーク自体に以下の対策が組み込まれている。

- **NULL対策**: 空の値による予期せぬエラーを防止する。
- **型チェック**: データ型の間違いを早期発見できる。
- **重複チェック**: 同じ名前の二重登録を防止する。
- **自動ロールバック**: エラー時のデータ不整合を防止する。

### 4.3 独自CSS（外部フレームワーク不使用）

画面表示に必要なデザイン部品も独自開発している。

- Bootstrap等の外部CSSフレームワークが不要である。
- 必要最小限のCSSのみで構成されている（1ファイルのみ）。
- レスポンシブに対応済みである（PC・タブレット・スマートフォン）。
- 学習コストが低い（覚えるデザイン要素が少ない）。

**効果**:
- 外部フレームワークのバージョンアップ対応が不要である。
- デザインの一貫性を保ちやすい。
- ページ読み込み速度が速い。

## 5. 開発コストへの効果

### 削減できる作業

| 作業 | 従来 | このフレームワーク |
|------|------|------------------|
| 画面デザイン変換 | 必要 | 不要 |
| URL設定 | 必要 | 不要 |
| データ変換コード | 必要 | 不要 |
| エラー処理実装 | 必要 | 最小限 |
| フレームワーク学習 | 長期間 | 短期間 |

### 習得期間の目安

| 対象 | 期間 |
|------|------|
| プログラマー | 1〜2週間程度である。 |
| AI（生成AI） | サンプルを与えれば即座に対応できる。 |

## 6. 適用領域

### 得意な分野

- 業務システム（データ登録・検索・更新・削除）に適している。
- 管理画面に適している。
- API提供（外部システム連携）に適している。
- マイクロサービス構成に適している。

### 特に効果が高いケース

- 中小規模のシステム開発。
- 短期間での開発。
- 少人数での開発・保守。
- AIを活用した開発。
